# 大千论坛 - 部署说明（Podman + PostgreSQL）

## 一、架构概览

```
用户浏览器 → Podman Pod (bbs-pod)
                 │
            Nginx (port 19848)
            ├── /bbs-user/   → bbs-ui 静态文件
            ├── /bbs-admin/  → bbs-admin-ui 静态文件
            └── /bbs-server/ → proxy_pass → Spring Boot JAR (port 8083)
                                               └── PostgreSQL (port 5432)
                                                     └── 数据目录: /data/sql/postgre
```

## 二、前提条件

- RHEL 7 服务器已安装 Podman（通过 `container-tools` 模块）
- ⚠ 前端已预构建，服务器**无需**安装 Node.js/npm
- JDK 8（开发环境已内置在 Docker 多阶段构建中）

## 三、交付包目录结构

```
部署/
├── bbs-server.jar                  ← Spring Boot 可执行 JAR（内嵌 Tomcat）
├── bbs-ui/                         ← 用户前端预构建静态文件（dist 内容直放）
├── bbs-admin-ui/                   ← 管理前端预构建静态文件（dist 内容直放）
├── docker/
│   ├── Dockerfile.bbs-server       ← 后端容器镜像构建文件
│   ├── Dockerfile.nginx            ← Nginx 容器镜像构建文件
│   └── nginx.conf                  ← Nginx 反向代理配置
├── scripts/
│   ├── bbs-pg-schema.sql           ← PostgreSQL 数据库表结构 DDL
│   ├── init-pg-database.sh         ← 数据库初始化脚本
│   └── migrate-mysql-to-pg.sh      ← MySQL→PG 数据迁移脚本
├── podman/
│   ├── create-pod.sh               ← 一键创建 Pod 脚本
│   ├── bbs-pod.service             ← systemd 服务单元
│   └── README.md                   ← Podman 部署详细指南
├── 环境配置/
│   ├── application.yml             ← 后端基础配置（含 context-path）
│   ├── application-podman.yml      ← Podman 环境数据库配置
│   ├── bbs-ui.env.production       ← 用户前端生产环境变量
│   └── bbs-admin-ui.env.production ← 管理前端生产环境变量
└── 部署说明.md                     ← 本文件
```

> **说明**：`bbs-ui/` 和 `bbs-admin-ui/` 中的内容是 `npm run build` 的 dist 输出结果直接展开，**没有嵌套 `dist/` 子目录**。这是因为交付包面向内网离线环境，已在开发机完成前端构建，服务器上无需 npm install。

## 四、部署步骤

### 4.1 容器镜像构建

将交付包上传到 RHEL 7 服务器后，在部署目录执行：

```bash
# 1. 构建后端镜像
podman build -t bbs-server:latest -f docker/Dockerfile.bbs-server .

# 2. 构建 Nginx 镜像（上下文为部署目录）
podman build -t bbs-nginx:latest -f docker/Dockerfile.nginx .
```

### 4.2 创建 Pod 并启动

```bash
# 设置数据库密码（生产环境请使用强密码）
export BBS_DB_PASSWORD='your_secure_password'

# 一键创建并启动 Pod（包含 PostgreSQL + BBS 应用 + Nginx）
chmod +x podman/create-pod.sh
./podman/create-pod.sh
```

### 4.3 数据库数据迁移

如果是新部署的 PostgreSQL，需要从旧 MySQL 迁移数据：

```bash
export MYSQL_PASSWORD='old_mysql_password'
export BBS_DB_PASSWORD='new_pg_password'

chmod +x scripts/migrate-mysql-to-pg.sh
./scripts/migrate-mysql-to-pg.sh
```

### 4.4 配置 systemd 开机自启

```bash
podman generate systemd --name bbs-pod --files --new
# 或使用预置服务文件
cp podman/bbs-pod.service /etc/systemd/system/
systemctl daemon-reload
systemctl enable bbs-pod.service
systemctl start bbs-pod.service
```

## 五、SELinux 配置（RHEL 7）

卷挂载需设置 SELinux 上下文：

```bash
semanage fcontext -a -t container_file_t "/home/asiayak/bbsUpload(/.*)?"
restorecon -Rv /home/asiayak/bbsUpload/

semanage fcontext -a -t container_file_t "/data/sql/postgre(/.*)?"
restorecon -Rv /data/sql/postgre/
```

## 六、验证

| 检查项 | 命令 |
|--------|------|
| Pod 状态 | `podman pod ps` |
| 后端健康 | `curl http://localhost:19848/bbs-server/` |
| 用户前端 | `curl http://localhost:19848/bbs-user/` |
| 管理前端 | `curl http://localhost:19848/bbs-admin/` |

## 七、端口信息

| 服务 | 端口 | 说明 |
|------|------|------|
| Nginx 对外入口 | **19848** | 生产环境通过此端口访问 |
| Spring Boot 后端 | 8083 | Pod 内部，仅 Nginx 可访问 |
| PostgreSQL | 5432 | Pod 内部，仅后端可访问 |

## 八、重要注意事项

1. **密码管理**：PostgreSQL 密码通过环境变量 `BBS_DB_PASSWORD` 注入，**严禁**硬编码在配置文件中
2. **数据持久化**：PostgreSQL 数据存储在 `/data/sql/postgre`，应用上传文件存储在 `/home/asiayak/bbsUpload/`，删除容器不影响这些数据
3. **旧数据中的硬编码 URL**：历史文章内容中可能包含 `http://192.168.3.36:8083/files/...` 等旧 IP 地址，迁移后这些图片链接将失效，需在数据库执行 UPDATE 替换域名
4. **离线构建说明**：
   - 本交付包中的前端已预构建完毕（`bbs-ui/` 和 `bbs-admin-ui/`），**服务器上无需安装 Node.js，也无需 `npm install`**
   - 如需修改前端源码并重新构建，必须在**有互联网的开发环境**中进行：
     1. 在项目根目录 `cd bbs-ui` 或 `cd bbs-admin-ui`
     2. 确保 `node_modules` 已安装（`npm ci` 或 `npm install`）
     3. 运行 `npm run build`（bbs-admin-ui 需要 `NODE_OPTIONS=--openssl-legacy-provider`）
     4. 将输出的 `dist/` 目录内容（而非目录本身）复制到本交付包对应的 `bbs-ui/` 或 `bbs-admin-ui/` 中覆盖
   - 本项目前端为 Vue 2，推荐 Node.js 18+，Node 24 可构建但有弃用警告
5. **交付包 Dockerfile 构建路径**：构建 Nginx 镜像时，上下文为 `部署/` 目录，命令为：
   ```bash
   podman build -t bbs-nginx:latest -f docker/Dockerfile.nginx .
   ```
   因此 Dockerfile 中的 COPY 路径是 `bbs-ui/`（而非 `bbs-ui/dist/`），路径已在交付包中适配好
