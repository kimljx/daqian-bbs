# 大千论坛 - 部署说明（Podman + PostgreSQL）

## 一、架构概览

```
用户浏览器 → Podman Pod (bbs-pod)
                 │
            Nginx (port ${NGINX_PORT}, 默认为 19848)
            ├── /bbs-user/   → bbs-ui 静态文件
            ├── /bbs-admin/  → bbs-admin-ui 静态文件
            └── /bbs-server/ → proxy_pass → Spring Boot JAR (port ${BBS_SERVER_PORT}, 默认为 8083)
                                               └── PostgreSQL (port ${BBS_DB_PORT}, 默认为 5432)
                                                     └── 数据目录: /data/sql/postgre
```

## 二、前提条件

- RHEL 7 服务器已安装 Podman（通过 `container-tools` 模块）
- ⚠ 前端已预构建，服务器**无需**安装 Node.js/npm
- JDK 8（开发环境已内置在 Docker 多阶段构建中）

## 三、交付包目录结构

```
daqian-bbs-offline/                  ← 离线部署包
├── images/
│   ├── bbs-server.tar               ← 后端容器镜像（podman save 导出）
│   ├── bbs-nginx.tar                ← Nginx 容器镜像（podman save 导出）
│   └── postgres-12.tar              ← PostgreSQL 12 镜像
├── podman/
│   ├── create-pod.sh                ← 一键创建 Pod 脚本（支持端口/路径配置）
│   ├── bbs-pod.service              ← systemd 服务单元
│   └── README.md                    ← Podman 部署详细指南
├── scripts/
│   ├── bbs-pg-schema.sql            ← PostgreSQL 数据库表结构 DDL
│   ├── init-pg-database.sh          ← 数据库初始化脚本
│   └── migrate-mysql-to-pg.sh       ← MySQL→PG 数据迁移脚本
├── deploy.sh                        ← 一键部署入口脚本
└── README.md                        ← 本文件
```

> **说明**：前端静态文件已预构建并包含在 nginx 容器镜像中，服务器上无需 Node.js/npm。

## 四、部署步骤

### 4.1 导入容器镜像

```bash
# 将镜像导入 Podman（离线环境，从 tar 文件加载）
podman load -i images/bbs-server.tar
podman load -i images/bbs-nginx.tar
podman load -i images/postgres-12.tar
```

### 4.2 配置环境变量

> **所有端口和路径都通过环境变量配置，部署人员可自由修改。**

```bash
# === 必填：数据库密码 ===
export BBS_DB_PASSWORD='r123456'

# === 端口配置（可选，有默认值） ===
export BBS_NGINX_PORT=19848      # Nginx 对外端口（宿主机映射端口）
export BBS_SERVER_PORT=8083      # Spring Boot 后端内部端口
export BBS_DB_PORT=15432         # PostgreSQL 端口（对应宿主机映射端口）

# === 路径配置（可选，有默认值） ===
export BBS_UPLOAD_DIR=/data/bbs/bbsUpload     # 上传文件存储目录
export PG_DATA_DIR=/data/sql/postgre           # PostgreSQL 数据目录
```

### 4.3 创建 Pod 并启动

```bash
chmod +x podman/create-pod.sh

# 方式一：PostgreSQL 使用**外部已有**数据库（不启动 PG 容器）
./podman/create-pod.sh

# 方式二：PostgreSQL 也在容器内运行
START_POSTGRES=1 ./podman/create-pod.sh
```

> **注意**：`create-pod.sh` 也支持 `--help` 参数查看所有可配置变量：
> ```bash
> ./podman/create-pod.sh --help
> ```

### 4.4 数据库数据迁移

如果是新部署的 PostgreSQL，需要从旧 MySQL 迁移数据：

```bash
export MYSQL_PASSWORD='old_mysql_password'
export BBS_DB_PASSWORD='r123456'

chmod +x scripts/migrate-mysql-to-pg.sh
./scripts/migrate-mysql-to-pg.sh
```

### 4.5 配置 systemd 开机自启

```bash
# 创建环境变量文件（所有配置写入此文件，供 systemd 加载）
cat > /etc/bbs-pod.env << EOF
BBS_DB_PASSWORD=r123456
BBS_NGINX_PORT=19848
BBS_SERVER_PORT=8083
BBS_DB_PORT=15432
BBS_UPLOAD_DIR=/data/bbs/bbsUpload
PG_DATA_DIR=/data/sql/postgre
START_POSTGRES=1
EOF

# 安装 systemd 服务
cp podman/bbs-pod.service /etc/systemd/system/
systemctl daemon-reload
systemctl enable bbs-pod.service
systemctl start bbs-pod.service
```

## 五、SELinux 配置（RHEL 7）

卷挂载需设置 SELinux 上下文：

```bash
semanage fcontext -a -t container_file_t "/data/bbs/bbsUpload(/.*)?"
restorecon -Rv /data/bbs/bbsUpload/

semanage fcontext -a -t container_file_t "/data/sql/postgre(/.*)?"
restorecon -Rv /data/sql/postgre/
```

## 六、验证

| 检查项 | 命令 |
|--------|------|
| Pod 状态 | `podman pod ps` |
| 后端健康 | `curl http://localhost:19848/bbs-server/` |
| 用户前端 | `curl http://localhost:19848/bbs-user/` |
| 管理前端 | `curl http://localhost:19848/bbs-admin/` |
| 数据库 | `podman exec bbs-postgres psql -U work_flow -d bbs -c "SELECT count(*) FROM bbs_article;"` |

> 注：以上 `19848` 为默认端口，如已修改 `BBS_NGINX_PORT`，请以实际值为准。

## 七、端口信息（默认值）

| 服务 | 端口 | 说明 |
|------|------|------|
| Nginx 对外入口 | **19848**（可配置） | 宿主机端口，通过 `BBS_NGINX_PORT` 修改 |
| Spring Boot 后端 | 8083（可配置） | Pod 内部，仅 Nginx 可访问，通过 `BBS_SERVER_PORT` 修改 |
| PostgreSQL | 5432（可配置） | Pod 内部（映射到宿主机），通过 `BBS_DB_PORT` 修改 |

## 八、端口配置详解

所有端口均通过环境变量控制，Spring Boot 和 Nginx 配置在容器启动时动态渲染。

**配置优先级：**
1. 环境变量 `BBS_NGINX_PORT`、`BBS_SERVER_PORT`、`BBS_DB_PORT`
2. 未设置时使用脚本内默认值

**典型场景：**

```bash
# 场景 1：使用默认端口
export BBS_DB_PASSWORD='r123456'
./podman/create-pod.sh

# 场景 2：Nginx 使用 80 端口（标准 HTTP 端口，需 root 权限）
export BBS_NGINX_PORT=80
export BBS_DB_PASSWORD='r123456'
./podman/create-pod.sh

# 场景 3：PostgreSQL 使用 15432 端口
export BBS_DB_PORT=15432
export BBS_DB_PASSWORD='r123456'
./podman/create-pod.sh

# 场景 4：全自定义端口 + 路径 + 容器内 PostgreSQL
export BBS_NGINX_PORT=80
export BBS_SERVER_PORT=9090
export BBS_DB_PORT=15432
export BBS_UPLOAD_DIR=/data/bbs/bbsUpload
export PG_DATA_DIR=/data/sql/postgre
export BBS_DB_PASSWORD='r123456'
START_POSTGRES=1 ./podman/create-pod.sh
```

## 九、重要注意事项

1. **密码管理**：PostgreSQL 密码通过环境变量 `BBS_DB_PASSWORD` 注入，**严禁**硬编码在配置文件中
2. **数据持久化**：PostgreSQL 数据存储在 `/data/sql/postgre`，应用上传文件存储在 `/data/bbs/bbsUpload/`，删除容器不影响这些数据
3. **旧数据中的硬编码 URL**：历史文章内容中可能包含 `http://192.168.3.36:8083/files/...` 等旧 IP 地址，迁移后这些图片链接将失效，需在数据库执行 UPDATE 替换域名
4. **离线构建说明**：
   - 本交付包中的前端已预构建完毕并打包进 Nginx 镜像，**服务器上无需安装 Node.js**
   - 如需修改前端源码并重新构建，必须在**有互联网的开发环境**中进行：
     1. 在项目根目录 `cd bbs-ui` 或 `cd bbs-admin-ui`
     2. 确保 `node_modules` 已安装（`npm ci` 或 `npm install`）
     3. 运行 `npm run build`（bbs-admin-ui 需要 `NODE_OPTIONS=--openssl-legacy-provider`）
     4. 重新构建 Nginx 镜像
   - 本项目前端为 Vue 2，推荐 Node.js 18+，Node 24 可构建但有弃用警告
5. **离线镜像构建**：如需要在离线服务器上重新构建镜像，需先在**有互联网的开发机**上用 `podman build` 构建并 `podman save` 导出，再到服务器 `podman load`
6. **Python 版本说明**：服务器默认 Python 2.7.5，另有 Python 3.10 可用。部署脚本基于 bash，不依赖 Python
